1) application_analysis.ipynb - анализ данных из основого файла application_train.csv.
2) bureau_agg_analysis.ipynb - аггрегирваоние данных из бюро про прошлый кредиты клиентов. В итоге сохранённый файл с добавленными признаками.
3) previous_applications_agg_analysis.ipynb - аггрегирваоние данных о прошлых обращениях клаентов в Credit Home. В итоге созранённый файл в новыми призанками.
4) all_data_agg.ipynb - аггрегирвоание всех ранее полученных датасетов в один со всеми признаками из всех файлов. Разделение на train/val/test и сохранение файлов.
5) preprocessing.ipynb - обработка данных. Датасет со всеми призанками: удалили признакми с большим количеством пропусков. Обработали выбросы. Заполнили пропущенные занчения. Обработали категорильные признаки. Убрали сильно коррелированные признаки.
Сохранили чистые без пропусков датасеты, готовые для применениия к модели.
